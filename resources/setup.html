If you're seeing this message, you have the app jar running on the server!

However, before we get started, there are a few things you need to check:

  • PostgreSQL
    • Did you install postgres?
    • Did you create a schema with the correct name?
    • Did you set up the password in the environmental variable?
  • Nginx
    • Did you install Nginx?
    • Did you configure it as a reverse proxy?
    • Did you get an encryption cert from letsencrypt?
    • Did you set up HTTP to HTTPS redirect in Nginx config?

If you didn't set up nginx yet, thats ok but do it after this. Supreme Reality expects a reverse proxy of some sort, and IP Bans will not work without that. Also, it's [current year] and there is no excuse for not using encryption.

If you have no idea what I'm talking about, go back and read the documentation (README.md)

If you at least have postgres set up and working, press 'next' to proceed

If you get the white screen of death, or a 500 error, or any other error screen at this step, it means you didn't set up postgres right. You may have to restart the app, fix postgres and try again.
